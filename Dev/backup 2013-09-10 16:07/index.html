I make websites! Want one?

HTML (Hyper Text Markip Language) is the language that defines the structure and layout of a web page. HTML5 is the upcoming new standard, replacing HTML4 and XHTML, and most of the features are already supported by modern browsers. It does cool things like support svg (click on the middle circle for an example of what that can do)

JavaScript is a scripting language used to make web pages interactive. It runs on the visitor's computer and doesn't require constant downloads from the website. It is also the scripting language for the Qt UI framework, and many other applications. JQuery is a JavaScript library that makes it easy to interact with a web page.

Django is a high-level Python Web framework. It manages the databases, and allows users to update the content easily, while giving the developer complete control over how the content is manipulated and displayed.

CSS (Cascading Style Sheets) is the language for defining the appearance of a web page. CSS3 is the new standard, and includes fun new effects such as shadows, rounded corners, transformations and animations

Qt is an open-source cross-platform application framework built on C++. The front-end development (Qt Quick) uses a language called QML (Qt Markup Language) that is conceptually similar to a combination of HTML and CSS, and makes use of JavaScript. It allows for rapid prototyping and creation of functional interactive user interfaces that can be deployed accross most platfroms.

MySQL (Structured Query Language) is an open source relational database management system that runs as a server, providing multi-user access to a number of databases. It is a popular database system for web pages.

    The basics of search engine optimisation (SEO)

    SEO is a huge topic, and companies invest huge amounts of money and effort researching and developing ways to improve their rankings on search engines. This makes it seem a very intimidating and intractable topic to hopeful young web developers like myself. But actually the basics are pretty simple, and mostly common sense.

    Most importantly, and obviously, you need to make sure your HTML, CSS and JavaScript code is valid and following best practices. You could try W3C’s validation tools, Lint, and JSHint to check your site.

    Firstly: design for users, not for search engines. There's been a kind of arms race between search engines and websites trying to increase traffic in any way they can. For example in the beginning search engines used helpful keyword information embedded in the html <meta> tags to determine the content of websites, but malicious websites would use that to promote their website by stuffing it with popular keywords that were nothing to do with the content of the website, and so the search engines had to get smarter. They are now very good at matching what uses are looking for to the actual content of a site, while also penalising sites that try to manipulate them.

    Secondly: search engines read your website as plain text, they can't see the amazing graphics, JavaScript special effects, or CSS that you might have created to lead the user around the page and highlight important information. (there are also some users who will prefer to view websites this way too). If you only had the HTML to your page, would your site still be understandable by a user? Have a look at your website through tools like seo-browser.com to try this out.

    1. Make sure all images have useful alt attributes to display when the images can't be seen.
    2. Your HTML should have a good structure: make sure you are making correct use of heading tags <h1> to <h6>, and in HTML5 the new structural tags such as <heading>, <section>, and <article>.
    3. Tip: Images can go in <h1> elements, and the alt text is treated as the heading by search engines, this way you can have your fancy heading graphic, while also maintaing the document structure to be picked up by search engines.

    Thirdly. Lets talk about keywords. Keywords are just as important for a user browsing your webpage looking for information, as they are for search engine rankings. Again, if you design for the user, the search engines will take care of themselves.

    Fourthly: what appears in search engine results is the title and meta description of your page, so make sure these are sensible to ensure people will click you when you do appear in search engines.
